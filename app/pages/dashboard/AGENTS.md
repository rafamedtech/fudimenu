# Dashboard rules

- Priorizar claridad sobre densidad visual.
- Usar Nuxt UI antes de crear componentes manuales.
- Los formularios deben ser simples.
- Mantener navegación consistente entre restaurantes, categorías y platillos.
- Cualquier acción destructiva debe ser explícita.
- No mezclar lógica pesada de datos directamente en componentes si puede ir en server routes o composables.
- La experiencia debe funcionar bien en móvil y escritorio.
